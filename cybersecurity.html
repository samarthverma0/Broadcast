Cyber Security 101




What is Cyber Security?



Also referred to as information security, Cybersecurity is the body of technologies, practices, and processes which are designed to protect networks, devices, programs, and data from attack, damage, or unauthorized access over the internet.

Global organizations are turning to human-centric cybersecurity, a new approach that places focus on changes in user behavior instead of an exponential number of growing threats.


Importance of Cyber Security



The world relies on technology more than ever before. There are a greater number of devices than people living in the world.

Today, businesses and governments store a large amount of that data on computers and transmit it across networks to other computers.

Devices and their underlying systems have vulnerabilities that, when exploited, undermine the health and objectives of an organization.

A data breach can have a range of devastating consequences for anyone.

It can decline a company’s reputation through the loss of trust.

The loss of critical data, such as source files or intellectual property or sensitive files, can cost a company or an individual its competitive advantage.

Data breach is just one consequence of cyberattack, but you need to be secure from other kinds of cyberattacks such as:

* Spoofing, spying, Malicious software, Trojans, Malwares etc.

* High-profile data breaches make media headlines.

* Thus, to be secure from cyberattacks and data breaches It is essential that organizations adopt and implement a strong cybersecurity approach.



The Scale of cyber threat:



Today, cyberattacks are increasing at a rapid pace despite record levels of security spending.





Types of cybersecurity



These are the types of cybersecurity, namely:

Network Security :



Network security is the practice of preventing and protecting against unauthorized access and intrusion at network level.

Network security focuses on how devices interact and on the connection source between them.

It is the process of taking physical and software preventative measures to protect the underlying networking infrastructure.



Application Security



Application security is the process of making applications more secure by finding, fixing, and enhancing the security in them.

Most of it is done at the development phase of application.




Information Security



Information Security is basically the practice of preventing unauthorized access, use, disclosure, disruption, modification, inspection, recording or destruction of information.

Information can be any data that makes sense like your profile on social media, your data in mobile phone, your biometrics, etc.

Information Security covers many research areas like Cryptography, Mobile Computing, Cyber Forensics, Online Social Media etc.


Data security



Data security is a set of standards and technologies that protect data from destruction, modification or disclosure.

Data security can be applied using a range of techniques and technologies, including administrative controls, physical security, logical controls, organizational standards, and other safeguarding techniques that limit access to unauthorized or malicious users or processes.

Data security technology comes in many shapes to protect data from a growing number of threats.

Ways of securing data include:

  • Data encryption
  • Data masking
  • Hardware security module
  • Deletions & erasure
  • Authentication
  • Access control
  • Tokenization



Database Security



Database security encircle a range of security measures taken to protect the Database files stored in a DBMS software.

Database security encompasses multiple controls:

  • Including system hardening
  • Access
  • DBMS configuration
  • Security monitoring


These different security controls help to define the security protocols.


Infrastructure Security




The security measures to protect infrastructure, especially critical infrastructure, such as network communications, communication center, server center, database center, and IT center.


Cloud security



Cloud security is the protection of data stored online via cloud computing platforms from theft, leakage, and deletion.

Methods of providing cloud security include:

  • Firewalls
  • Penetration testing
  • Obfuscation
  • Tokenization,
  • Virtual Private Networks (VPN)
  • Avoiding public internet connections.



Mobile security



Mobile security involves protecting information stored and transmitted from smartphones, tablets, laptops and other mobile devices.

The term mobile security is a broad one that covers protecting mobile devices from malware threats to reducing risks and securing mobile devices and their data in the case of theft, unauthorized access or accidental loss of the mobile device.


Data Loss Prevention (DLP)



Data Loss Prevention (DLP) is the practice of detecting and preventing data breaches or unwanted destruction of sensitive data.

Organizations use DLP to protect and secure their data and comply with regulations.

Organizations typically use DLP to:

  • Protect Personally Identifiable Information (PII)
  • Protect Intellectual Property critical for the organization
  • Achieve data visibility in large organizations
  • Secure mobile workforce and enforce security in Bring Your Own Device (BYOD) environments
  • Secure data on remote cloud systems




Common types of cyber threats



Malware:



Malware is a term used to describe malicious software,

It includes spyware, ransomware, viruses, and worms.

Malware breaches a network or a system through a vulnerability, when a user installs a risky software containing malware,

malware can do the following:

  • Blocked access to the network (ransomware)
  • Obtains information by transmitting data from the hard drive (spyware)
  • Disrupts the systems working by destroying data and files.



Phishing:



Phishing is the practice of sending fraudulent communications that appear to come from a reputable source, usually through email.

The goal is to steal sensitive data like credit card and login information or to install malware on the victim’s machine.

Phishing is an increasingly common cyberthreat.


Man-in-the-middle attack:



Man-in-the-middle (MitM) attacks, also known as eavesdropping attacks.

It is a general term for when an attacker positions himself in a conversation between a user and an application or user and a user.

The goal of this attack is to steal personal information, such as login credentials, account details and credit card numbers.

Targets are the users of financial applications, SaaS businesses, e-commerce sites and other websites where logging in is required.

When a MitM attack is done it appear as a normal exchange of information to the user.

Two common points of entry for MitM attacks:

  • On unsecure public Wi-Fi, attackers can insert themselves between a visitor’s device and the network. Without knowing, the visitor passes all information through the attacker.
  • Once malware has breached a device, an attacker can install software to process all of the victim’s information.



Denial-of-service attack (DOS):



A denial-of-service attack floods systems, servers, or networks with traffic to exhaust resources and bandwidth.

As a result, the system is unable to response legitimate requests.

When attackers use multiple compromised devices to launch this attack. This is known as a Distributed-Denial-of-Service (DDoS) attack.


SQL injection:



A Structured Query Language (SQL) injection occurs when an attacker inserts malicious code into a server that uses SQL and forces the server to reveal information it normally would not.

An attacker does a SQL injection simply by submitting malicious code into a vulnerable website search box.


Zero-day exploit:



A zero-day exploit occurs after a network vulnerability is announced but before a solution is implemented.

Attackers target the announced vulnerability during this window of time.


Cross-site scripting (XSS) attack:




XSS attacks use third-party web resources to run scripts in the victim’s web browser or scriptable application.

The attacker injects a backdoor with malicious JavaScript into a website’s database.

When the victim requests a page from the website, the website transmits the page, with the attacker’s backdoor as part of the HTML body, to the victim’s browser, which executes the malicious script.

These vulnerabilities can give attacker full control of the victim's computer.




Managing cybersecurity



Cyber Security should be properly managed for the growth and good health of an organization

Underlying are the steps taken to manage cybersecurity:

  • Identify and evaluate risks
  • Assess ability to shoulder risk
  • Develop and implement risk reduction measures
  • Implement, monitor and revise
  • Disclose risks and strategies



What about me?



A simple question you will always ask, what about my own privacy ?

For this we have various minimal measures can be taken to be secure:

Think before you click



  • Just because you can click, doesn’t mean you should.
  • Always remember it can cost you your privacy and data.
  • Malicious links can do damage in many ways, so be sure to inspect links and ensure they’re from trusted senders before clicking.



Use Two-Factor Authentication



  • Two factor authentication provides two layers of security measures so if an attacker can accurately guess your password, there is still an additional security measure in place to ensure that your account is still safe.



Look Out for Phishing Scams



  • phishing attacks are some of the greatest cybersecurity threats as they are very easy to fall for.
  • The best way to be safe from phishing scams is by avoiding emails from unfamiliar senders, look for grammatical errors or any inconsistencies in the email that looks suspicious, and hover over any link you receive to verify what the destination is.



Keep it strong



  • Always try to use strong and distinctive passwords, which a person can't guess easily.



Keep Up with Updates



  • Updates can be issued when security flaws are discovered.
  • If you find these update notifications to be annoying, you’re not alone.
  • But you can consider them the lesser of two evils when weighing up rebooting your device versus putting yourself at risk for malware and other types of computer infection.



Connect Securely



  • Stick to private networks when possible, especially when handling sensitive information and making payments.
  • Secure Your Mobile Device
  • Security doesn’t end at your desktop.
  • It’s important to get into the habit of securing your mobile device as well.
  • Use strong passwords and biometric features, ensure you turn off your Bluetooth, don’t automatically connect to any public Wi-Fi, and download with caution.
  • Stick to antivirus software.



Back-Up Your Data



  • There’s no excuse not to have a backup of important data.
  • Back it up on a physical location and on the cloud.
  • Malicious threats and hackers don’t always want to steal your data, but sometimes their end-aim is to encrypt or erase it.
  • Back it up to have an ultimate recovery tool.
